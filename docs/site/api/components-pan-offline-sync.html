pan-offline-sync - Offline-first synchronization component

Class: PanOfflineSync

pan-offline-sync - Offline-first synchronization component
Queues mutations when offline and automatically syncs when back online.
Provides conflict resolution and retry logic with exponential backoff.

Attributes

Attribute Type Description
storage string IndexedDB database name for queue storage
retry-max number Maximum retry attempts (default: 3)
retry-delay number Initial retry delay in ms (default: 1000)
topics string Comma-separated topic patterns to queue
strategy string Conflict resolution: 'server-wins' | 'client-wins' | 'merge'
debug boolean Enable debug logging

Examples

<pan-offline-sync
  storage="offline-queue"
  retry-max="3"
  retry-delay="1000"
  topics="todos.*,users.*"
  debug>
</pan-offline-sync>
